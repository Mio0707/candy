# 葫芦狮模型拆件操作指南

当前模型 `模型/葫芦狮-原件.glb` 的情况是：1 个整体网格、1 个贴图材质。它不能靠程序准确自动拆成“头、脚、胡子”等语义部件，必须在 Blender 里做人工归类。

好消息是：模型内部有很多自然断开的几何小块，可以先按“松散部件”分离，再把小块归并成 9 个网页对象。

## 1. 最终要得到的 9 个对象

导出的 `.glb` 里必须有这些对象名：

```text
base
body_core
front_legs
back_legs
head
tail_ears
back_mustache
head_lines
ball
```

网页只认这些名字。名字不对，代码就找不到对应部件。

## 2. 推荐流程

### 第一步：用脚本生成拆件工作台

1. 打开 Blender。
2. 进入 `Scripting` 面板。
3. 打开或粘贴 `tools/blender/prepare_loose_parts.py`。
4. 点击运行。

脚本会做这些事：

- 导入 `模型/葫芦狮-原件.glb`。
- 把整体模型按松散部件分成很多 `loose_001`、`loose_002` 这样的小对象。
- 建好 9 个 `PART_...` 集合。
- 保存 `模型/葫芦狮-拆件工作台.blend`。

这一步像是先把一大块糖塑拆成很多已经松开的“小糖块”，还没有判断每块属于哪里。

### 第二步：把小对象拖进对应集合

在 Blender 右上角 Outliner 里，把 `loose_xxx` 小对象拖到对应集合：

| 集合 | 放什么 |
|---|---|
| `PART_base` | 红色葫芦底座 |
| `PART_body_core` | 黑色身体主体，不含脚 |
| `PART_front_legs` | 前脚 |
| `PART_back_legs` | 后脚 |
| `PART_head` | 红色头部、大眼、大嘴、牙齿、舌头 |
| `PART_tail_ears` | 尾巴、耳朵、飘带类装饰 |
| `PART_back_mustache` | 黄色背部糖衣、胡子 |
| `PART_head_lines` | 红绿糖条、头部和嘴部线条 |
| `PART_ball` | 爪子下方圆球 |

如果某块拿不准，先放到最接近的集合。第一版重点是能分阶段显示，不需要像医学切片一样精确。

### 第三步：每个集合合并成一个对象

对每个 `PART_...` 集合重复：

1. 在 Outliner 里选中这个集合里的所有小对象。
2. 按 `Ctrl + J` 合并。
3. 把合并后的对象重命名为目标对象名，例如 `base`。

最后场景里应该只剩下 9 个主要网格对象。

### 第四步：运行命名检查

在 Blender 的 `Scripting` 面板运行：

```text
tools/blender/validate_parts.py
```

如果提示“拆件命名检查通过”，说明对象名对了。

### 第五步：导出 glb

在 Blender 里选择：

```text
File > Export > glTF 2.0
```

建议设置：

```text
Format: glTF Binary (.glb)
Include: Selected Objects 或 Visible Objects
Transform: +Y Up 默认即可
Geometry: Apply Modifiers 可勾选
Materials: Export
Images: Automatic
```

导出文件建议命名：

```text
模型/葫芦狮-手动拆件.glb
```

## 3. 快速判断怎么归类

如果你不知道一个小块属于哪一类，可以用这个判断：

- 它是承重的大红色底座：`base`
- 它是黑色大身体：`body_core`
- 它在身体下面、靠前：`front_legs`
- 它在身体下面、靠后：`back_legs`
- 它有眼睛、嘴、牙齿、舌头：`head`
- 它像外展的耳朵、尾巴、飘带：`tail_ears`
- 它是黄色背部大装饰或胡子：`back_mustache`
- 它是细条、红绿线、嘴部边线：`head_lines`
- 它是爪子下方圆球：`ball`

## 4. 如果时间很紧

先只拆 5 大块也可以：

```text
base
body_core
head
tail_ears
ball
```

然后临时复制或合并出剩余对象：

- `front_legs` 和 `back_legs` 可以先从身体附近拆粗一点。
- `back_mustache` 可以先和 `tail_ears` 类似处理。
- `head_lines` 可以先只放最明显的红绿糖条。

第一版要的是“每一步有东西出现”，不是每根糖条都完美分离。
