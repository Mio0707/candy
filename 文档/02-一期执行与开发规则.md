# 一期执行与开发规则

本文件给开发和建模使用，所有规则服务于“吹糖造物”电脑端 Demo。

## 1. 技术建议

一期建议使用：

- React + Vite：搭建网页。
- Three.js 或 React Three Fiber：加载和控制瑞兽三维模型。
- MediaPipe Hands：识别摄像头里的手势。
- Blender：制作并导出 `.glb` 模型。

技术原则：

- 模型动画由网页控制。
- 手势识别只判断基础动作，不追求复杂识别。
- 所有手势步骤都有按钮兜底。
- 页面先保证电脑端可展示。
- 摄像头失败时，仍然可以用按钮完整走完流程。

## 2. 页面结构

一期页面保持简单，不做复杂官网。

建议包含：

1. 标题区：显示“吹糖造物”和副标题。
2. 三维展示区：显示瑞兽模型，是页面核心。
3. 摄像头预览区：显示用户摄像头画面，可以小一些。
4. 当前步骤提示：一句话告诉用户当前要做什么。
5. 备用按钮：统一文案“继续造物”。
6. 非遗说明：放在页面底部，用短句说明灵感来源和边界。

建议提示文案：

| 阶段 | 提示文案 |
|---|---|
| 开场 | 准备好，一起吹出你的糖塑瑞兽。 |
| 吹底座 | 握拳三次，把糖团慢慢吹起。 |
| 放身体 | 握拳取糖块，再捏住放到底座上。 |
| 拉四肢 | 捏住向下拉，拉出瑞兽的脚。 |
| 安头部 | 握拳取头部糖块，再捏住安到身体前方。 |
| 贴装饰 | 跟随提示完成尾巴、耳朵、糖衣、糖条和圆球。 |
| 选气运 | 用手掌停在一个气运上。 |
| 托起瑞兽 | 用手掌托住瑞兽，完成加持。 |
| 完成 | 你的糖塑瑞兽已加持。 |

## 3. 交互状态

建议状态顺序：

```text
intro
camera_ready
base_step
body_step
legs_step
head_step
decor_step
fortune_step
lift_step
complete
```

状态机规则：

- 当前步骤不匹配时，事件不能触发动画。
- 动画播放中锁定按钮，避免重复触发。
- 每一步完成后自动进入下一步。
- 重置时所有部件回到初始状态。
- 摄像头输入和备用按钮都必须走同一套状态机。

## 4. 核心手势规则

| 阶段 | 手势 | 触发结果 |
|---|---|---|
| 吹起底座 | 任意一只手握拳，必须“握拳 - 松开 - 再握拳” | 计数 3 次，底座从 20% 到 45%、75%、100% |
| 放置身体 | 先握拳，再拇指食指捏合并向底座方向移动 | `body_core` 出现并吸附到底座 |
| 拉出四肢 | 捏合后向下移动，分两次 | 先出现 `front_legs`，再出现 `back_legs` |
| 安放头部 | 先握拳，再捏合并向身体前方移动 | `head` 出现并安放到身体前方 |
| 尾巴和耳朵 | 捏合后向下移动 | `tail_ears` 出现 |
| 背部糖衣和胡子 | 手掌张开后向下移动 | `back_mustache` 出现 |
| 头部和嘴部糖条 | 单根手指横向移动 | `head_lines` 出现 |
| 圆球 | 双手靠近，再捏合向爪子下方移动 | `ball` 出现并落位 |
| 选择气运 | 手掌悬停在光球区域约 1 秒 | 选中对应气运 |
| 托起瑞兽 | 手掌在瑞兽下方停留约 1 秒 | 瑞兽上浮并播放粒子 |

## 5. 模型拆分规则

用户负责用 Blender 建模，并导出一个 `.glb` 文件。

要求：

- 导出一个完整 `.glb` 文件，不给多个零散文件。
- 视觉上是一只完整瑞兽。
- 内部部件必须保持独立对象。
- 每个部件都摆在最终正确位置。
- 不需要用户制作动画。
- 不需要单独制作小糖块。
- 颜色建议在 Blender 中先上好基础色，网页可以做少量颜色和发光调整。

模型对象命名必须统一：

| 模型名称 | 内容 | 初始状态 |
|---|---|---|
| `base` | 红色底座 | 显示，缩小 |
| `body_core` | 黑色身体主体，不含四肢 | 隐藏 |
| `front_legs` | 两只前脚 | 隐藏 |
| `back_legs` | 两只后脚 | 隐藏 |
| `head` | 头部、大眼睛、大嘴、牙齿、舌头等 | 隐藏 |
| `tail_ears` | 尾巴 + 两个耳朵 / 飘带装饰 | 隐藏 |
| `back_mustache` | 黄色背部糖衣 + 胡子 | 隐藏 |
| `head_lines` | 头部和嘴巴上的糖条 | 隐藏 |
| `ball` | 爪子下方圆球 | 隐藏 |

## 6. 动画规则

动画要像糖料被无形力量塑造，不出现真实手、人物或工具。

建议表现：

- 有弹性和糖料鼓起感。
- 不要太夸张卡通。
- 保留实体糖塑的重量感。
- 每一步动画约 0.6 到 1.2 秒。
- 主要使用显示、隐藏、缩放、移动、发光和粒子。

## 7. 气运规则

| 气运 | 代码名 | 颜色 | 含义 | 粒子效果 |
|---|---|---|---|---|
| 顺 | `shun` | 绿色 | 顺遂 | 绿色粒子向上流动 |
| 喜 | `xi` | 黄色 | 喜乐 | 黄色粒子跳动散开 |
| 勇 | `yong` | 黑金 | 镇护 | 黑金粒子聚拢环绕 |
| 旺 | `wang` | 红色 | 兴旺 | 红色粒子围绕圆球旋转 |

## 8. 核心模块接口建议

为了方便合并，核心交互代码建议暴露：

```js
initBeastExperience({
  stageContainer: '#beast-stage',
  videoElement: '#camera-video',
  modelUrl: '/models/beast.glb',
  onStepChange: (step) => {},
  onComplete: () => {}
})
```

建议方法：

```js
startExperience()
nextAction()
selectFortune(type)
destroyExperience()
```

说明：

- `startExperience()`：开始摄像头和三维体验。
- `nextAction()`：备用按钮调用，推进当前步骤。
- `selectFortune(type)`：备用按钮选择气运。
- `destroyExperience()`：离开页面时清理摄像头和动画。

## 10. 兜底和错误提示

必须包含：

- 摄像头不可用时，按钮模式仍然可用。
- 模型加载失败要显示错误。
- 缺少模型部件名要显示错误。
- 动画不要过度闪烁。
- 所有按钮有明确文字。
- 重置按钮可以回到初始状态。

模型缺少部件时建议提示：

> 模型缺少必要部件：xxx。请检查 Blender 对象命名。
