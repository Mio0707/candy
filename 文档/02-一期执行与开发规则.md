# 一期执行与开发规则

本文件给开发和建模使用，所有规则服务于“吹糖造物”电脑端 Demo。

## 1. 技术建议

一期建议使用：

- React + Vite：搭建网页。
- Three.js 或 React Three Fiber：加载和控制瑞兽三维模型。
- MediaPipe Hands：识别摄像头里的手势。
- Blender：制作并导出 `.glb` 模型。

技术原则：

- 模型动画由网页控制。
- 手势识别只判断基础动作，不追求复杂识别。
- 所有手势步骤都有按钮兜底。
- 页面先保证电脑端可展示。
- 摄像头失败时，仍然可以用按钮完整走完流程。

## 2. 页面结构

一期页面保持简单，不做复杂官网。

建议包含：

1. 标题区：显示“吹糖造物”和副标题。
2. 三维展示区：显示瑞兽模型，是页面核心。
3. 摄像头预览区：显示用户摄像头画面，可以小一些。
4. 舞台手势提示：在模型下方用无文本框的短句提示当前动作；使用方正粗黑宋简体及楷体回退、暖白糖色和轻微阴影。
5. 备用按钮：统一文案“继续造物”。
6. 非遗说明：放在页面底部，用短句说明灵感来源和边界。

运行模式：

- `/model-stage/` 为正式体验，隐藏摄像头参数、骨骼标记、校准、步骤跳转、显示全部和部件检查。
- `/model-stage/?debug=1` 为调试模式，保留完整开发工具，并提供返回正式体验的入口。
- 模式切换只改变界面可见内容，不改变手势判定和造物流程。

建议提示文案：

| 阶段 | 提示文案 |
|---|---|
| 开场 | 准备好，一起吹出你的糖塑瑞兽。 |
| 吹底座 | 张开手掌准备，再握拳吹起底座。 |
| 放身体 | 再次握拳，让身体出现并自动落位。 |
| 拉四肢 | 合拢后逐渐张开拇指和食指，拉出前后脚。 |
| 贴糖衣 | 张开后逐渐并拢拇指和食指，贴上背部糖衣。 |
| 出现头部 | 张开手掌，让头部糖块在空中出现。 |
| 安放头部 | 握拳，让头部落到身体前方。 |
| 出现尾巴耳朵 | 合拢后逐渐张开拇指和食指，让尾巴和耳朵糖片出现。 |
| 贴尾巴耳朵 | 张开后逐渐并拢拇指和食指，让尾巴和耳朵贴合。 |
| 贴糖条 | 合拢后逐渐张开拇指和食指，贴上红绿糖条。 |
| 出现圆球 | 张开手掌，让圆球糖料出现。 |
| 安放圆球 | 握拳，让圆球落到爪下。 |
| 旋转展示 | 瑞兽完成后自动旋转一周，无需额外手势。 |
| 托起瑞兽 | 用手掌托住瑞兽，完成祝福。 |
| 收下祝福 | 等待粒子显形后握拳，收下祝福。 |
| 完成 | 你的糖塑瑞兽已收到祝福。 |

## 3. 交互状态

建议状态顺序：

```text
intro
camera_ready
base_step
body_step
legs_step
back_decor_step
head_form_step
head_place_step
tail_ears_form_step
tail_ears_place_step
head_lines_step
ball_form_step
ball_place_step
showcase_step
lift_step
fortune_reveal_step
complete
```

状态机规则：

- 当前步骤不匹配时，事件不能触发动画。
- 手势判定优先参考当前步骤：握拳步骤优先识别完整拳头，两指步骤才优先识别拇指和食指并拢，避免紧拳被误判为捏合。
- 握拳连续稳定约 0.16 秒后触发，不依赖动作切换瞬间的单帧结果。
- 手势未达到完成阈值前，模型按动作进度实时预览变化；达到阈值后吸附定型。
- 动画播放中锁定按钮，避免重复触发。
- 每一步完成后自动进入下一步。
- 重置时所有部件回到初始状态。
- 摄像头输入和备用按钮都必须走同一套状态机。

| 12 | 出现圆球 | 空中糖料位置 | 张开手掌 | 圆球跟随张掌程度逐渐出现 | 点“继续造物” | 圆球暂不落位 |

## 5. 核心手势规则

| 阶段 | 手势 | 触发结果 |
|---|---|---|
| 吹起底座 | 张掌后逐渐握拳 | 底座跟随握拳程度鼓起，达到阈值后定型 |
| 放置身体 | 松拳后再次逐渐握拳 | `body_core` 跟随握拳程度变大并靠近底座，最后吸附 |
| 拉出四肢 | 单手拇指和食指逐渐张开 | `front_legs` 和 `back_legs` 从 `body_core` 内部跟随两指张开程度实时延展 |
| 贴背部糖衣 | 单手拇指和食指逐渐并拢直至指尖接触，食指允许自然弯曲 | `back_mustache` 跟随两指并拢并吸附到身体 |
| 出现头部 | 逐渐张开手掌 | `head` 跟随张掌程度在空中成形，暂不落位 |
| 安放头部 | 张掌后逐渐握拳 | `head` 跟随握拳程度落到身体前方 |
| 出现尾巴和耳朵 | 单手拇指和食指逐渐张开 | `tail_ears_1` 和 `tail_ears_2` 跟随两指张开程度在空中出现 |
| 贴尾巴和耳朵 | 单手拇指和食指逐渐并拢 | `tail_ears_1` 和 `tail_ears_2` 跟随两指并拢程度贴合 |
| 贴糖条 | 单手拇指和食指逐渐张开 | `head_lines` 跟随两指张开程度贴合 |
| 出现圆球 | 逐渐张开手掌 | `ball` 在空中跟随张掌程度成形 |
| 安放圆球 | 张掌后逐渐握拳 | `ball` 跟随握拳程度落到爪下 |
| 旋转展示 | 无手势 | 整件作品自动旋转一周并随机生成祝福 |
| 托起瑞兽 | 在瑞兽上方张掌准备，保持张掌向下移动，再在瑞兽下方稳定停留约 0.55 秒 | 瑞兽随手掌逐渐上浮，确认稳定托住后触发祝福粒子 |
| 收下祝福 | 手掌由张开变成握拳 | 祝福粒子从瑞兽轮廓散开并向手心方向汇聚，表现“祝福被收下” |

- 祝福粒子完整显形后，粒子瑞兽平滑跟随画面中的手掌中心移动；跟随响应速度为初版的 2 倍，跟随范围限制在舞台中央，短暂丢失手掌时保持当前位置。
- 粒子瑞兽跟随手掌移动期间，旋转速度为原速度的 2 倍。
- 用户握拳触发“收下祝福”后停止位置跟随，粒子消散和汇聚动画单独以原速度的 2 倍播放。
- 所有流程阶段统一使用开场的暖色摄像头蒙版，不在祝福阶段切换为暗色蒙版。

## 6. 模型拆分规则

用户负责用 Blender 建模，并导出一个 `.glb` 文件。

要求：

- 导出一个完整 `.glb` 文件，不给多个零散文件。
- 视觉上是一只完整瑞兽。
- 内部部件必须保持独立对象。
- 每个部件都摆在最终正确位置。
- 不需要用户制作动画。
- 不需要单独制作小糖块。
- 颜色建议在 Blender 中先上好基础色，网页可以做少量颜色和发光调整。

模型对象命名必须统一：

| 模型名称 | 内容 | 初始状态 |
|---|---|---|
| `base` | 红色底座 | 显示，缩小 |
| `body_core` | 黑色身体主体，不含四肢 | 隐藏 |
| `front_legs` | 两只前脚 | 隐藏 |
| `back_legs` | 两只后脚 | 隐藏 |
| `head` | 头部、大眼睛、大嘴、牙齿、舌头等 | 隐藏 |
| `tail_ears_1` | 尾巴 | 隐藏 |
| `tail_ears_2` | 两个耳朵 | 隐藏 |
| `back_mustache` | 黄色背部糖衣 | 隐藏 |
| `head_lines` | 头部和嘴巴上的糖条 | 隐藏 |
| `ball` | 爪子下方圆球 | 隐藏 |

## 7. 动画规则

动画要像糖料被无形力量塑造，不出现真实手、人物或工具。

建议表现：

- 有弹性和糖料鼓起感。
- 不要太夸张卡通。
- 保留实体糖塑的重量感。
- 每一步动画约 0.9 到 1.8 秒；手势识别成功后先完成当前造型动画，再进入下一步。
- 主要使用显示、隐藏、缩放、移动、发光和粒子。
- 前脚和后脚不能突然显现，也不能从上方出现，要从黑色 `body_core` 内部方向向下长出。
- 圆球分两段表现：先随张掌在空中聚成形，再随握拳落到爪下。
- 从贴尾巴和耳朵开始，糖条、圆球和完成状态都保持斜正面角度，不再自动转回侧面。
- 旋转展示时整件作品只绕竖直方向平行旋转一圈，不上下翻滚，时长约 3.6 秒，结束后回到展示角度。
- 托起瑞兽时模型整体轻微上浮，实体糖塑变半透明。
- 祝福显形时使用“实体糖塑 + 粒子轮廓”：粒子大致沿瑞兽表面分布，颜色根据随机祝福变化。
- 收下祝福时粒子先轻微散开，再向画面下方手心方向汇聚，表现“祝福被用户收进手里”。

## 8. 祝福规则

| 祝福 | 代码名 | 颜色 | 含义 | 粒子效果 |
|---|---|---|---|---|
| 顺 | `shun` | 绿色 | 顺遂 | 绿色粒子向上流动 |
| 喜 | `xi` | 黄色 | 喜乐 | 黄色粒子跳动散开 |
| 勇 | `yong` | 黑金 | 镇护 | 黑金粒子聚拢环绕 |
| 旺 | `wang` | 红色 | 兴旺 | 红色粒子围绕圆球旋转 |

## 9. 核心模块接口建议

为了方便合并，核心交互代码建议暴露：

```js
initBeastExperience({
  stageContainer: '#beast-stage',
  videoElement: '#camera-video',
  modelUrl: '/models/beast.glb',
  onStepChange: (step) => {},
  onComplete: () => {}
})
```

建议方法：

```js
startExperience()
nextAction()
generateRandomFortune()
destroyExperience()
```

说明：

- `startExperience()`：开始摄像头和三维体验。
- `nextAction()`：备用按钮调用，推进当前步骤。
- `generateRandomFortune()`：旋转展示结束后随机生成并固定祝福。
- `destroyExperience()`：离开页面时清理摄像头和动画。

## 11. 兜底和错误提示

必须包含：

- 摄像头不可用时，按钮模式仍然可用。
- 模型加载失败要显示错误。
- 缺少模型部件名要显示错误。
- 动画不要过度闪烁。
- 所有按钮有明确文字。
- 重置按钮可以回到初始状态。

模型缺少部件时建议提示：

> 模型缺少必要部件：xxx。请检查 Blender 对象命名。
